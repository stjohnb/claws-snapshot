# Codebase scanning expansion

### Committed secrets and credentials scanner (#669)

Add a daily scanning job (matching the existing scanner pattern) that scans all cloned repos for accidentally committed secrets. Check tracked files for: AWS access keys (`AKIA...`), GitHub tokens (`ghp_`, `gho_`, `ghs_`), generic API keys (high-entropy strings assigned to variables named `*_KEY`, `*_SECRET`, `*_TOKEN`), private SSH/PGP keys (`-----BEGIN`), database connection strings with embedded passwords, and .env files that shouldn't be in version control. Use regex patterns inspired by gitleaks/truffleHog but implemented as a lightweight Node.js scanner (no external dependency). For each repo, scan the current HEAD of the default branch plus the last 50 commits (via `git log -p`) to catch recently-committed-and-reverted secrets that remain in history. Create a deduped alert issue per repo (title: 'Alert: Potential secrets detected in repository') with a table of findings: file path, line number, secret type, and a masked preview. Exclude known false positives (test fixtures, example configs with placeholder values). Does not require Claude — purely filesystem/git scanning with regex matching.
